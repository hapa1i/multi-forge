# accidental_complexity_cleanup -- remove dead scaffolding + drift-prone duplication

**Lane**: `proposed/` -- accepted-candidate batch, not yet scheduled. The items are **independently shippable**; move
individual slices (or the whole batch) to `doing/` as they are picked up. This is not blocking any other work.

**Origin**: full-codebase simplicity audit, 2026-07-01 (`/simplicity_audit --full`). 15 parallel first-principles
auditors (one per feature area, each pointed at its own design-doc sections for Justification-Search step 0), then a
per-finding **adversarial verifier** that tried to *refute* each finding by searching consumers repo-wide, reading git
history, and checking tests/design docs. 33 candidates -> **20 survivors** (13 refuted as load-bearing). Every
high-impact survivor was then independently re-verified against code (file:line) before this card was written.

**Addendum (manual review pass, 2026-07-01)**: item **#21** (stale CLI-alias claims) was added from a maintainer review
and code-verified; item **#14** (legacy search) was tightened from the audit's original "delete + repoint oracle" to the
lower-risk "privatize + self-contained characterization test" scope. Both trace-verified against the current checkout.

**Type**: Single **batch** card, deliberately **not an epic**. The items are mostly independent deletions that share a
*theme* (accidental complexity), not a shared contract, sequencing decision, or code seam (`board_contract.md` epic
test). Two small coupled pairs are noted under [Sequencing & coupling](#sequencing--coupling).

**Scope / branch note**: the audit ran against `main` @ `5973d602`; that commit was **docs-only** (the consumer_lanes
epic closeout), so the *source* it examined is byte-identical to this branch's base (`9bf60b8c`). All anchors below were
re-verified on the current checkout. Two items live in files another branch is actively editing -- see
[Sequencing & coupling](#sequencing--coupling).

**References**: `docs/developer/coding_standards.md` §5 (research-preview clean-break + durable-state rules -- the
policy most deletions rely on); `docs/developer/cli_style_guidelines.md` (the shared-logic rule Proposal 1 violates);
`docs/board/impl_notes.md` (the intentional-decision record the refuted findings leaned on).

---

## Why (the audit's thesis)

Multi-Forge's complexity is **overwhelmingly essential or earned** -- the audit produced **84 do-not-simplify callouts**
vs 20 removals, and the adversarial pass refuted 39% of candidates as load-bearing. The accidental complexity that
survived has a clear signature and is worth removing precisely because it is *not* over-abstraction:

1. **Initial-commit scaffolding.** `git log -S` on nearly every dead item points to the single `3e0a636e` "Initial
   release" commit -- speculative generality (`resolve_template_paths`, `load_yaml_strict`, `ConfigSecretsProvider`,
   `_coerce_env_value`, native gemini/openai proxy providers) that the project never used and never grew.
2. **Parallel-runtime / parallel-plane duplication.** Recent copy-paste that entered when a second runtime or telemetry
   plane was added *beside* the first without hoisting the shared part (Claude+Codex invokers, downstream+provider-trace
   literals, CLI+skill templates). These are **live drift hazards** -- two copies that must be hand-synced.

Removing these banks the cleanup while the intent is still fresh; the earned complexity (durable-state discipline,
fail-open composition, the runtime seam, wire translation) is explicitly left alone.

### Surfaced defects (not simplifications -- bugs the audit found)

The adversarial pass turned up real defects while trying to *justify* complexity. These are **behavior/observability
bugs**, not cleanups -- decide separately whether to fix here or file:

- **Defect B (confirmed, High).** The proxy **auth-retry success path** (`proxy/server.py:1607-1653`) makes a real
  provider call and records cost + metrics but **never calls `record_provider_trace`** -- the only two trace call-sites
  are the non-retry success paths (`:1394`, `:1478`). A request that 401s -> refreshes credentials -> succeeds on retry
  produces cost/metrics with **no provider-trace record** -- exactly the "what happened to this request?" gap
  provider-trace exists to close (its origin incident was a supervised fork through OpenRouter). Fix: add a
  `record_provider_trace(...)` call on the retry branch (the source-capability gate is inside the helper, so calling
  unconditionally is safe).
- **Gap A (open question, needs audit).** `cli/policy.py` `supervisor_evaluate` decides exit code from **prose-prefix
  matching** (`_INFRA_FAILURE_PREFIXES`) only, while `PolicyDecision.fail_open` exists and `engine.py` checks
  `fail_open OR prose`. If any fail-open path sets `fail_open=True` without a matching warning prefix,
  `forge policy supervisor evaluate` reports `passed=true`/exit-0 on a fail-open. Needs an exhaustive check of fail-open
  emitters before deciding whether it is a real gap.
- **Minor C.** `_parse_consensus_worker_specs` lacks the `code_mode` param its debate sibling has. Impact is marginal
  (explicit named roles validate against a flat `NAMED_ROLES` dict regardless of mode), so it is an inconsistency to
  note, not a user-visible bug.

## Do not simplify (the balance)

The audit produced **84 do-not-simplify callouts** -- kept here as a pointer so future cleanup does not re-litigate
them. Leave alone (Essential/Earned, each cross-referenced to a design doc / impl_note):

- **Durable-state & fail-open discipline**: three-layer override validation, `StateCorruptedError` vs
  `StateUnreadableError`, atomic `temp+os.replace+fsync`, strict versioned deserialization, `gc.py` fail-closed
  detection, best-effort telemetry writes that never raise, `PolicyEngine` fail-open composition.
- **Money/telemetry correctness**: the three physically-separate planes joined by run tree, idempotent
  `downstream_event_id` merge, durable cap checkpoint + bootstrap reconciliation, shadow at-most-once billing.
- **Runtime seam & consumer-lanes**: `_HeadlessLifecycleBase` template split, `lanes.py` resolver, the consumer-lane
  TOCTOU freeze re-check under lock, T7 degrade overlay stored separately from the frozen binding.
- **Session lifecycle**: the 5-way resume dispatch tree, codex-runtime dispatch *before* Claude predicates, the deferred
  same-dir-fork resume-id resolver.
- **Wire/security**: `converters.py` (~1300 lines of hard translation), redaction-before-persistence ordering,
  `stream_relay` once-only teardown, fail-closed launcher identity gates.

The refutation also **preserved** the `cli-session` module split + its `_sess()`/wildcard re-export layer (documented
consequence of the 2.5K-line limit + 255 test patch targets) and the `CredentialManager` class itself (only two of its
methods are unused, #17).

## Acceptance (per-item assertion pattern)

When a slice is executed, its checklist item is ticked only when: (a) `git grep` of the removed symbol returns only the
deletion, (b) the focused test module passes, and (c) for `--json`/wire/durable-state touches (#9, #10, #16), a test
pins the new shape. Defect B needs a regression test asserting the auth-retry path emits a provider-trace record.
